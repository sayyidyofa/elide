Package elide.server.assets

See: Description

  • Class Summary 
    Class Description
    AssetControllerBuilt-in controller implementation which bridges the configured asset serving prefix to the active AssetManager layer for this server run.
    AssetPointerReference to an application-embedded asset.
    AssetReferenceRepresents a resolved reference to an asset at serving-time, before it is rendered into a link or other tag.
    RenderedAssetIntermediary class which represents an asset that has been fully prepared to serve to an end-user request, including any headers which should apply to the response.
    ScriptDescribes a JavaScript asset which is embedded in a given Elide application, and described by Elide's protocol buffer structures; when read from the application bundle and interpreted, this class is used to hold script info.
    StylesheetDescribes a stylesheet asset which is embedded in a given Elide application, and described by Elide's protocol buffer structures; when read from the application bundle and interpreted, this class is used to hold document info.
    TextDescribes a generic text asset of some kind, for example, humans.txt or robots.txt; when read from the app bundle and interpreted, this class is used to hold file info.
    ServerAssetDescribes a server-side asset which is embedded in an application bundle through Elide's asset tools and protocol buffer for asset bundle metadata.
    ServerAssetManagerBuilt-in asset manager implementation for use with Elide applications.
    ServerAssetReaderDefault implementation of an AssetReader; used in concert with the default AssetManager to fulfill HTTP requests for static assets embedded within the application.
  • Interface Summary 
    Interface Description
    AssetManagerDescribes the API surface of an Asset Manager, which controls access to embedded server-side assets such as CSS and JS documents.
    AssetManifestLoaderDefines the API surface of a loader which is capable of searching for an asset manifest at server startup, and then de-serializing the manifest, so it can be indexed and then made available via an AssetManager implementation.
    AssetReaderDescribes the API surface expected for a reader of static assets from some data source; responsible for efficiently reading assets from a resource path and producing resulting content.
    AssetResolverDescribes the expected API surface for a resolver of server-side assets, which, in cooperation with an AssetReader and under the management of an AssetManager, is responsible for checking asset paths for existence and translating them to absolute paths which may be read and served.
